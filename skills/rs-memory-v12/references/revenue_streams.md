### Bug Bounty Workflow

```
1. SCOPE ANALYSIS
   ├── Read program scope carefully
   ├── Identify in-scope assets
   ├── Note out-of-scope items
   └── Check disclosure policy

2. RECONNAISSANCE (40% of time)
   ├── Subdomain enumeration
   ├── Directory bruteforcing
   ├── Technology fingerprinting
   ├── API endpoint discovery
   └── Parameter discovery

3. VULNERABILITY TESTING (40% of time)
   ├── Automated scanning (Burp, Nuclei)
   ├── Manual testing (OWASP Top 10)
   ├── Business logic testing
   ├── IDOR testing
   └── Authentication/Authorization testing

4. REPORTING (20% of time)
   ├── Clear description
   ├── Step-by-step reproduction
   ├── Impact assessment
   ├── Video/screenshot evidence
   └── Remediation suggestions
```

### Report Template

```markdown
# [Vulnerability Title]

## Summary
Brief description of the vulnerability and its impact.

## Steps to Reproduce
1. Navigate to [URL]
2. Click on [element]
3. Enter [payload]
4. Observe [result]

## Impact
- What can an attacker achieve?
- What data is exposed?
- What systems are affected?

## Evidence
[Screenshots/Video]

## Remediation
How to fix this vulnerability.

## References
- CWE-XXX
- OWASP Category
```
